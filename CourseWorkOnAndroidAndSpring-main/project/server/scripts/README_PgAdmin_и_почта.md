# Почта не приходит и в PgAdmin «ничего нового»

## Почему в PgAdmin только 7 пользователей и старые данные

1. **Приложение могло запускаться не с той БД.**  
   По умолчанию активен профиль **H2** (локальная файловая БД). В ней создаются свои данные, а в PostgreSQL в PgAdmin — только то, что однажды накатила миграция V1.

2. **Чтобы новые данные появились в той же БД, что и в PgAdmin:**
   - Запускайте приложение с профилем **postgres**:  
     `-Dspring.profiles.active=postgres` или переменная окружения `SPRING_PROFILES_ACTIVE=postgres`.
   - В `application-postgres.yml` (или переменных) укажите **тот же** URL БД, что открыт в PgAdmin:  
     `spring.datasource.url=jdbc:postgresql://localhost:5432/student_performance` (хост/порт/имя БД как у вас).
   - После первого успешного запуска с postgres Flyway применит миграции V2–V9, появятся новые пользователи (student4…student13), студенты, оценки и т.д.

3. **Если миграции уже применялись к другой БД**, в текущей в PgAdmin по-прежнему будет только начальный набор. Тогда либо переключите приложение на эту БД (тот же URL), либо выполните скрипты миграций вручную к этой БД.

## Почему на почту ничего не приходит

Письма отправляются **на тот email, который записан в таблице `users`** у того пользователя, под которым вы вошли (или для которого создаётся отчёт). Сейчас там тестовые адреса (`admin@mi.vlgu.ru`, `student1@mi.vlgu.ru` и т.д.) — это не реальные ящики, письма на них не доставляются.

**Что сделать:**

1. **Указать в БД реальный email** для учётки, под которой тестируете (например, admin):
   - Откройте в PgAdmin скрипт `scripts/update_user_emails_for_mail.sql`.
   - Замените `YOUR_REAL_EMAIL@mail.ru` на **вашу настоящую почту** (ту, к которой есть доступ).
   - Выполните скрипт в Query Tool к **той же базе**, к которой подключается приложение.

2. **Настроить отправителя (Gmail — тот же что у админа):**
   - По умолчанию в приложении уже задан хост `smtp.gmail.com` и порт 587.
   - Нужно указать **логин и пароль приложения** одним из способов:
     - **Переменные окружения:** `MAIL_USER=ваш@gmail.com`, `MAIL_PASSWORD=пароль_приложения`  
       (в IntelliJ: Run → Edit Configurations → выберите ServerApplication → Environment variables).
     - **Локальный конфиг:** скопируйте `application-local-example.yml` в `src/main/resources/application-local.yml`, подставьте свой Gmail и пароль приложения, добавьте в `spring.profiles.active` профиль `local` (например: `postgres,local`).
   - **Пароль приложения Google:** [Аккаунт Google](https://myaccount.google.com/) → Безопасность → Двухэтапная аутентификация (включить) → Пароли приложений → создать пароль для «Почта». Этот пароль подставьте в `MAIL_PASSWORD` или в `application-local.yml`.
   - Адрес отправителя (From) будет совпадать с `MAIL_USER` (ваш Gmail).

3. Перезапустить приложение и снова войти под admin (или под тем пользователем, чей email вы обновили). Письма при входе и при создании отчёта будут уходить на этот адрес.

**Проверка:** зайдите под админом в раздел «Проверка почты» (`/api/web/admin/test-email`) и отправьте тестовое письмо на тот же email — так можно убедиться, что SMTP настроен и письмо не уходит в спам.

## Краткий чек-лист

| Шаг | Действие |
|-----|----------|
| 1 | Запуск с `spring.profiles.active=postgres` и URL БД как в PgAdmin |
| 2 | В PgAdmin выполнить `update_user_emails_for_mail.sql`, подставив свой email |
| 3 | В приложении задать `spring.mail.host`, `username`, `password` |
| 4 | Перезапуск приложения, вход под admin, проверка входа и раздела «Проверка почты» |
